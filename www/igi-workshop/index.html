Okay, but what is this "Clouduboy" thing?


Note: the animation shows an earlier version, the UI, features and functionality may have somewhat improved and/or changed since this was recorded

Clouduboy is not a thing, but rather, a variety of things. It's a set of tools, libraries and apps I have been working on in the past few years, trying to figure out the best way to bring software (buzzword: coding) and hardware/microcontrollers (or in 2017 terms, "IoT") closer to people. Read about the components below, or scroll further down to see what's in it for you if you come to the workshop.

HTML5 game creation library

Clouduboy's games are written with the help of the MicroCanvas JavaScript game library. The library is an extension to the HTML5 Canvas API, and contains a few handy shortcuts to make game creation easier and lessen the need for boilerplate code. At the end of the day, it's perfectly possible to create games without the library as well, but the MicroCanvas library makes it easier and the code more readable.

To be able to compile/run your games on micrcocontrollers it is currently inevitable having to use the library/boilerplate provided.

Pixelart/sprite creation tools & library

Embedded in the MicroCanvas library is the PixelData library which lets you create pixelelgraphic assets and simple sprites/animations for your games without the need for external resources or a graphics editor – all you need is your text editor (or the Clouduboy IDE).

There is also a simple tune/sound effect format you could use to embed sounds/blips and blops in your games. When all said and done, a MicroCanvas game is just a single JavaScript file embedded in a webpage.

Microcontroller code compiler

After developing/debugging and sharing your game in the browser Clouduboy lets you create machine-code binaries for your games for various platforms. With this you (and anyone you share your games with) can not only play your games on-the-go on a variety of commercially available gaming platforms, but you could build your very own, hand-crafted compatible devices and experiment, even add extra features or sensors!

Fully web-based editor/IDE

For easier development the Clouduboy platform includes its own, in-browser editor. This means that (especially if you are developing to select platforms), the only thing you need is a web browser to create, play and even install games on microcontrollers. Bring your PC, Mac, Chromebook, Android or iOS tablet, even your phone (although we do recommend an external keyboard if you want to go this deep into the rabbithole :P), and create your games in the same, familiar UI.

Coding tutorials

One of the original goals of Clouduboy was to serve as a tool to help interested folks learn the basics of coding, while keeping them engaged, whether they are doing it in the form of self-learning or via mentored courses/workshops.

At the workshop you will get a chance to try the special Clouduboy coding-quick-start tutorials that are not released for the public yet. You will also get full access to these (and future) tutorials, should you want to continue learning after the workshop is over.

Who is this workshop for?

I know it's a trite to say "everyone", but if you are at least a slight bit interested in programming this workshop is for you. To avoid sounding vague, here are a few scenarios to consider – who are you?

I'm a code newbie

I'm new to computers and/or coding but were always interested to learn.
I'm a designer/UI/UX/IX person who have worked with developers, but didn't (have to) code, but interested to have a look.

Come to the workshop and…

  • …learn the basics of coding.
    You need to touch on a fair bit of the first principles when you set out to build a game – what is a variable, what are basic control structures like conditions and loops, what are functions, events and more!

  • …have fun!
    May sounds superfluous, but it makes sense actually: you learn best/fastest when you are having fun.

    And with Clouduboy, there's loads of fun to be had. ;)

  • …hack on hardware and learn!
    Learn aboutabout sensors, components, microcontrollers etc. and electronics basics. All this is enough to demistify computer software and hardware – if you can believe you can build it, you already won!

I'm a developer interested in games/hardware

I'm a developer who is already proficient in one or many (high or low level, doesn't matter) programming languages and curious to learn more about games programming, web development or hardware/electronics/IoT.

At the workshop you will get a chance to…

  • …get started with hardware with zero setup!
    Download & install the Arduino IDE, wait for compiling, fool around with unrecognized driver – forget all this. Bring any web-capable device, even a Chromebook or tablet (keyboard recommended), and just open the IDE in the browser. Write your code, test it. Flashing? Just download the compiled binary, plug in and copy over the file, that's it you are done!

  • …play games!!!
    But no, seriously, who doesn't love to do that? :D

  • …but more importantly (re-)create your own versions of classic games
    Or completely new ones. But doing your version of Space Invaders with invading Mercurean Muffins and a toothpaste defender to save Planet Dentin from doomed to destruction, or Snake but a twist (you play the food! portals!) has its own charm too.

  • …DIY everything, including the hardware!
    We'll help you cobble together a basic minuscule arcade machine, so it being your code – the game you built single-handedly is powering the device – the device itself was also built by you and you alone! Creation-ception!

The Chrome offline dino run game/easter egg, recreated for the original Arduboy DevKit and a custom, Arduino Micro-based gaming hardware.

I'm a developer interested in web development

I've already played around with < insert IoT device here > but frankly that didn't seem too exciting/useful – I'm a web developer who is proficient in JavaScript and wants to learn a few new tricks of the trade.

You are in luck! Why? Because…

  • …you can learn about advanced web APIs, like Canvas, WebAudio or Web Animations!
    Games are actually pretty complex beasts, fast-changing animated visuals and sounds react to inputs from the user, or the passing of time – you will need all the power of the web platform to get somewhere. Don't fret though, as…
  • …MicroCanvas is built to try not to reinvent the wheel.
    The libraries extend functionality of the built-in browser APIs here & there, like the HTML5 Canvas API, but you will learn about (and will be using) a good bunch of standard browser APIs that you can later use in any development scenario!
  • …MicroCanvas code supports ES2015+ features!
    MicroCanvas uses a subset of the full JavaScript API (if you want to retain the ability to compile for hardware) – but a subset of the new JavaScript standard, that means that a good bunch of new features of ES6 and later you can use in your games too!
  • …you will create complex games for tiny microcontrollers
    And learn about low-level languages and hardware in the process.
  • …finally you can "IoT" in a fun, engaging way!
    Forget "blinking LED" examples, you are creating something that actually makes sense, but using the same techniques and sneakily learning about sensors, electronics components etc. as you otherwise would.

On top of all this, you can share your games & creations online with anyone and play others' games on the tiny console you just built yourself. Because it's just the Web.

All this and the blissful satisfaction of having created your very own personal game, or having re-created timeless classics like Space Invaders or pocket-sized hit of the '97, Snake! You are taking away useful learnings of JavaScript, the web and electronics/microcontrollers, while not having to worry about falling asleep halfway through!

Hands-on, self-paced learning

This is a hands-on workshop, with a relatively short intro followed by mentored, self-directed learning. After the workshop is over you retain access to the IDE and everything else, so you can continue learning.

Although things may not be as polished as I want them to be just yet, the Clouduboy editor and tutorials already help you get started and to learn in your own pace. This is not one of those workshops when you lose track 30 minutes in and the whole day is wasted as for the rest of the day you are doomed to listen to babbling that's way over your head. Clouduboy guides you learning in your own pace, minimizing frustration and maximizing the fun! Got stuck? Have a stellar idea but don't know how to start? I'm there for you to help you out.

Notes

All the tools are free & open-source

All tools of the Clouduboy system are open source and available in the public GitHub repository. Most of the functionality is also in beta (or even alpha) phase. This means that not all (planned or potential) functionality may be available, but if you are a developer and this sounds interesting/exciting you can contribute and help out with development. This also means that all of this is provided as-is (e.g. some of the IDE infrastucture requires a server to run, which may get overwhelmed, DDOSed etc. and may not work. You can always run the tools on your local computer (since they are available online, and the code is open source), and we have secondary infrastucture set up to ensure that all goes smoothly during the workshops).

Hardware support

With the help of PlatformIO supports a variety of platforms based on Arduino, such as various 8-bit (Atmega32u4) and 32-bit (ARM) microcontrollers. Clouduboy has some support for commercial microcontroller-gaming platforms (Arduboy, Tiny Arcade), but provides guides for building your own devices as well.
